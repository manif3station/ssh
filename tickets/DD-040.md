# DD-040

## Title

Skip `ssh-add` when key fingerprint is already loaded.

## Method

TDD

## Problem

`ssh.list` can already show a key as `loaded`, but `ssh.add` was still calling `ssh-add` for that same key and prompting for the passphrase again. That breaks the expected idempotent workflow.

## Acceptance

- `ssh.add` does not call `ssh-add` for a key that is already loaded
- add output distinguishes newly added keys from already loaded keys
- Docker tests and coverage pass
- installed DD command proof passes
- latest DD source command proof passes
- release commit and push complete
