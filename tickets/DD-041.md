# DD-041

## Title

Add collector GUI askpass prompting and red-indicator exit behavior.

## Method

TDD

## Problem

The `ssh.door-opener` collector could report missing remembered keys, but it was not enforcing the expected user interaction policy. In desktop environments, users should get a passphrase dialog before they discover the problem during a real SSH connection. In non-GUI environments, the collector should fail visibly so DD can alert the user instead of silently staying green or hanging.

## Acceptance

- interactive collector runs still explain the passphrase request and call `ssh-add`
- GUI-capable non-interactive collector runs use an askpass-backed `ssh-add` path
- non-GUI non-interactive collector runs return collector status `missing` and a nonzero process exit
- Docker tests and coverage pass
- installed DD command proof passes
- latest DD source command proof passes
- release commit and push complete
