## Acceptance

- `dashboard ssh.add <key>` records a normalized key path in `config/ssh/keys.txt`
- key records are deduplicated
- a bare key name resolves to `~/.ssh/<name>`
- running without a key prefers existing `~/.ssh/id_rsa` and then `~/.ssh/id_ed25519`
- the command ensures a usable ssh-agent before calling `ssh-add`
